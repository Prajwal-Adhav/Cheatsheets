This is an Python CheatSheet for me!
  1. Installing pip
    pip is the reference Python package manager. It’s used to install and update packages. You’ll need to make sure you have the latest version of pip installed. Windows
    The Python installers for Windows include pip. You should be able to access pip using: 
    py -m pip --version
                        
    #pip 9.0.1 from c:\python36\lib\site-packages (Python 3.6.1)
  2. Creating a virtual environment
    virtualenv is used to manage Python packages for different projects. Using virtualenv allows you to avoid installing Python packages globally which could break system tools or other projects. You can install virtualenv using pip. On Windows: 
    py -m pip install --user virtualenv

  3. To create a virtual environment,

    go to your project’s directory and run venv. If you are using Python 2, replace venv with virtualenv in the below commands.
    On Windows: 
    py -m venv env
  4. Activating a virtual environment
    Before you can start installing or using packages in your virtual environment you’ll need to activate it. Activating a virtual environment will put the virtual environment-specific python and pip executables into your shell’s PATH. 
    .\env\Scripts\activate
    You can confirm you’re in the virtual environment by checking the location of your Python interpreter, it should point to the env directory.
    On Windows: 
    where python
                    .../env/bin/python.exe
    As long as your virtual environment is activated pip will install packages into that specific environment and you’ll be able to import and use packages in your Python application.

  5. Installing packages
    Now that you’re in your virtual environment you can install packages. Let’s install the Requests library from the Python Package Index (PyPI): 

    py -m pip install requests

  6. For installing Django

    pip install django

  7. For Upgrading or installing pip

    pip install --upgrade pip

  8. For Starting a new Django Project

    django-admin startproject project-name

  9. To remove a folder or file from Django Project

    rm folder/file-name

  10. To run the Django Project

    Go to the directory where the manage.py is located and type.. 
    python manage.py runserver

  11. If you get this warning while running the Django Project

    python manage.py migrate

  12. To get the username and password for logging into the Django Project admin panel

    Create username and password for admin login 
    python manage.py createsuperuser
  13. Sometimes when you create a new terminal in vs-code you have to first Activate the python environment to virtual if it exists
    Create username and password for admin login 
    python manage.py createsuperuser

  14. To start an app 

    python manage.py startapp app-name

  15. To ask Django to create a table for us in an app 

    python manage.py makemigrations
